# UniLodge v2 - Modern Student Accommodation Platform

A **complete, production-ready** platform for student accommodation discovery and management. This project is a modern, scalable application built with **Next.js**, **Node.js/Express**, and **MongoDB Atlas**, featuring a fully integrated **AI Engine**.

## 🎯 Overview

**Status**: Version 2.0.0 - Fully Operational ✅

This project implements a comprehensive UniLodge platform:

- ✅ **100% TypeScript** - Full type safety across the monorepo
- ✅ **MongoDB Atlas Database** - Scalable NoSQL document architecture
- ✅ **Custom JWT Authentication** - Secure, role-based access control (Guest, Admin, Warden)
- ✅ **Node.js AI Engine** - Integrated RAG (Retrieval-Augmented Generation) assistant using `@xenova/transformers`
- ✅ **Dynamic Interactive Dashboard** - Features Google Maps integrations and generated QR passes (Identity/Mess)
- ✅ **Admin Override System** - Admins can dynamically upload custom Maps and QR codes via LocalStorage

## 🏗️ Architecture

UniLodge v2 uses a three-tier microservice architecture housed in an NPM workspace monorepo:

### 1. Frontend (`apps/frontend` - Port 3000)
- **Framework**: Next.js (App Router)
- **Styling**: Tailwind CSS + Lucide React
- **Features**: 11 unique pages, role-based routing, interactive dashboards, dynamic SVG QR generation.

### 2. Backend API (`apps/backend` - Port 3001)
- **Framework**: Node.js + Express
- **Database**: MongoDB Atlas via Mongoose
- **Auth**: Custom JWT (JSON Web Tokens) with Auth Middleware
- **Features**: 9 comprehensive services handling Users, Bookings, Rooms, Universities, and system health.

### 3. AI Engine (`apps/ai-engine` - Port 3002)
- **Framework**: Node.js + Express
- **AI Models**: OpenRouter API integration (Minimax model) with local embeddings via `@xenova/transformers` (all-MiniLM-L6-v2)
- **Features**: In-memory RAG Vector Store seeded with knowledge documents, conversational chat endpoint, and smart university recommendations.

## 🚀 Quick Start

Ensure you have Node.js (v18+) installed.

1. **Install Dependencies**:
   ```bash
   npm install --legacy-peer-deps
   ```

2. **Environment Variables**:
   Ensure `.env` files are configured in all three app directories:
   - `apps/frontend/.env.local`
   - `apps/backend/.env`
   - `apps/ai-engine/.env`
   *(Refer to the `.env.example` files in each directory)*

3. **Start the Services**:
   Open three separate terminal windows and run:
   
   **Terminal 1 (Frontend):**
   ```bash
   cd apps/frontend
   npm run dev
   ```
   
   **Terminal 2 (Backend):**
   ```bash
   cd apps/backend
   npm run dev
   ```
   
   **Terminal 3 (AI Engine):**
   ```bash
   cd apps/ai-engine
   npm run dev
   ```

> [!WARNING]
> Do **NOT** attempt to run the AI Engine with Python/Uvicorn (e.g., `python -m uvicorn`). The AI Engine was successfully migrated to a pure Node.js Express server. Running the Python command will result in a 503 Service Unavailable error across the platform!

## ✨ Key Features

### 🔑 Authentication Flow
The system utilizes a secure JWT-based authentication flow. Tokens are generated by the backend and passed to the frontend, establishing secure sessions without relying on external third-party providers like Supabase.

### 🤖 AI Assistant (RAG Pipeline)
The `ai-engine` features a fully operational Retrieval-Augmented Generation pipeline. It embeds system documents into an in-memory vector store upon startup. When users ask questions in the frontend dashboard, the engine retrieves relevant context and passes it to the OpenRouter API to generate highly accurate, context-aware responses.

### 📱 Dynamic Guest Dashboard
- **Campus Map**: Integrates interactive Google Maps embeds based on the assigned University.
- **Digital Passes**: Automatically generates unique SVG QR codes for Identity Verification and Mess Hall Payments.
- **Admin Overrides**: Administrators can use the new Settings tab to upload custom images for maps and QR codes, which are encoded to Base64 and stored, seamlessly overriding the dynamic defaults.
